## 📐 Architecture

<div align="center">

```
╔══════════════════════════════════════════╗
║           PRESENTATION LAYER            ║
║    Compose UI  ·  ViewModel  ·  State   ║
╠══════════════════════════════════════════╣
║              DOMAIN LAYER               ║
║   Use Cases  ·  Entities  ·  Logic      ║
╠══════════════════════════════════════════╣
║               DATA LAYER                ║
║  Repository  ·  Room  ·  Retrofit  ·  FS║
╚══════════════════════════════════════════╝
      Dependency direction: inward only
```

</div>

<details>
<summary><sub>Constraints I apply on every project</sub></summary>

<br/>

| Constraint | Reason |
|---|---|
| No business logic in ViewModel | Use Cases own domain logic |
| No framework imports in Domain | Testable in isolation |
| Repository behind interface | Swappable, mockable data sources |
| Sealed classes for UI state | Exhaustive handling by compiler |

</details>
