---

<details>
<summary>Architecture model</summary>

<br/>

```
Presentation  ←  Compose UI · ViewModel · UI State
     ↓
   Domain     ←  Use Cases · Entities · Business Logic
     ↓
    Data      ←  Repository · Room · Retrofit · Firebase

Dependency direction: inward only.
```

**Constraints applied on every project**

- No business logic in ViewModel — Use Cases own domain
- No framework imports in Domain — testable in isolation
- Repository behind interface — swappable, mockable
- Sealed classes for UI state — exhaustive handling by compiler

</details>
